QA Engineer Onboarding

前三個月職能培養圖表

這份圖表把新人培養拆成三個月：第 1 個月先會照既有案例穩定執行， 第 2 個月開始補基本測試點，第 3 個月嘗試接低風險小模組，但仍由資深 QA review。

01 第 1 個月

建立基礎

建立 QA 基礎與產品熟悉

目標不是獨立負責，而是學會 QA 的基本工作方式：能照案例執行、留下正確證據、遇到不清楚的地方會標註。

定位：照做與記錄

他要會

  • 看懂 PRD / UI mockup 的基本結構。
  • 知道 Test Case 欄位與執行方式。
  • 會截圖、錄影、記錄實際結果。
  • 會寫基本 Bug report。
  • 知道 Happy Path：正常流程。
  • 知道 Exception Handling：例外處理。
  • 知道 Boundary / Edge Case：邊界條件。
  • 知道 Permission / State / Integration：權限、狀態、串接。
  • 知道 UI / UX / Content：畫面、文案、Mockup 對版。
  • 知道測試環境、帳號、測試資料不能亂用。

適合任務

  • 跟著資深 QA 跑 Smoke Test。
  • 跟著資深 QA 跑回歸測試。
  • 跟著跑權限驗證。
  • 幫忙整理測試證據。
  • 把口頭測試步驟整理成 Markdown / Excel。

DoD

  • 可以照測試案例完整執行。
  • Bug 回報不缺重現步驟、實際結果、預期結果、截圖。
  • 不清楚的地方會標註，不會自行補需求。
本月重點

先建立 QA 工作紀律：照案例執行、正確記錄、不要把推測當需求。

能力進展
照做 補測試點 小模組 Owner
02 第 2 個月

開始設計

開始寫簡單 Test Case

目標是從「照做」進到「可以補基本測試點」，能依 PRD / 畫面補出可執行、可 review 的案例初稿。

定位：補基本測試點

他要會

  • 依照 PRD / 畫面補 Happy Path 正常流程。
  • 補基本 Exception Handling，例如必填未填、格式錯誤、無權限。
  • 補基本 UI copy / layout 檢查。
  • 協助整理 Test Report 的測試結果。
  • 知道 Passed / Failed / Skipped 的判斷方式。

適合任務

  • 登入 / 驗證碼 / 時間篩選器等小功能。
  • L4 營運績效頁面中固定顯示檢查。
  • 協助維護 Stage 1 Smoke Test Case。

DoD

  • 可以寫出可執行的 Test Case。
  • Steps 清楚，Expected Result 能對應步驟。
  • 測試結果能被 QA / RD 讀懂。
  • 需要資深 QA review，但不是從零重寫。
本月重點

讓新人開始具備測試設計感，但先限制在低風險、規格清楚的小功能。

能力進展
照做 補測試點 小模組 Owner
03 第 3 個月

嘗試負責

開始接小模組 QA Owner

目標是讓新人負責一個低風險模組，能產出 Test Plan / Test Case 初稿、執行測試並整理風險，但放行判斷仍需 review。

定位：低風險 Owner

他要會

  • 產出小模組 Test Plan 初稿。
  • 產出小模組 Test Case 初稿。
  • 執行測試並追蹤 Bug。
  • 整理測試結果與未解風險。
  • 知道哪些問題要升級給資深 QA 判斷。
  • 開始理解 API / RBAC / 資料驗證基本概念。

適合任務

  • 登入流程。
  • 權限驗證中的單一角色或單一頁面。
  • L4 營運績效中的單一區塊。

DoD

  • 可以獨立跑完一個低風險模組。
  • 可以整理測試結果與未解風險。
  • 遇到需求不清、資料異常、權限不符時會回報。
  • 還不要求他獨立決定放行。
本月重點

開始培養 ownership，但風險判斷與放行標準仍由資深 QA 把關。

能力進展
照做 補測試點 小模組 Owner
